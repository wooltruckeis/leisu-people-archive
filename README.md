# leisu-people-archive

本项目是一个用于归档和发布多个独立 HTML 页面的静态仓库。  
仓库中存放的内容以 HTML 文件形式存在，不针对任何特定域名或网站。  
所有页面均以本地可打开、可浏览的格式保存，方便查阅与分发。

## 目录结构

```
.
├── pages/            # 存放所有独立 HTML 页面
├── assets/           # 页面引用的静态资源（图片、样式、脚本等）
├── index.html        # 可选：仓库首页或导航入口
└── README.md         # 本文件
```

- `pages/` 目录下每个子文件夹或文件对应一个独立归档页面。
- `assets/` 中资源按页面或类型组织，避免路径冲突。

## 页面归档说明

- 每个 HTML 页面均为独立存档，不依赖外部在线服务（CDN 资源除外，但建议尽量内联或本地化）。
- 页面内容可能包含文本、图片、样式及少量脚本，均以离线可用为优先目标。
- 文件名和路径应保持清晰，避免特殊字符，推荐使用英文或拼音命名。

### 归档维护说明

- 新增页面：在 `pages/` 下创建新目录或文件，并同步更新 `index.html` 中的导航链接（如有）。
- 更新页面：直接替换或修改对应 HTML 文件，注意保持引用资源路径正确。
- 删除页面：移除对应文件及链接，并清理 `assets/` 中不再使用的资源。
- 不建议在页面中嵌入外部跟踪代码、广告或动态加载的非必要内容。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接双击 `index.html` 或 `pages/` 下的任意 HTML 文件，即可在浏览器中打开浏览。
3. 如需部署到静态托管服务（如 GitHub Pages），将仓库根目录作为站点根目录即可。

## 注意事项

- 本仓库仅用于归档和展示，不提供在线服务或实时更新。
- 页面内容可能随时间变化，请以仓库中最新版本为准。
- 如发现页面显示异常，请检查资源路径或浏览器兼容性设置。
